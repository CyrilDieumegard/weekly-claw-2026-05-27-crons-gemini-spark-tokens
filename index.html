1. Les Crons sur OpenClaw

Les crons transforment un agent local en assistant proactif : il peut se réveiller à heure fixe, vérifier une source, résumer, relancer, publier une alerte ou préparer un livrable.

Exact timingRappels, routines, checks quotidiens, tâches hebdo, suivi de signaux.
Agent isoléUne tâche peut tourner sans polluer la session principale ni dépendre du contexte du chat.
DeliveryLe résultat peut revenir dans Discord, dans une session dédiée ou rester silencieux.
“La vraie bascule : passer de ‘demande-moi’ à ‘je surveille pour toi’.”

Démo : une routine utile en 5 minutes

Objectif : créer une automatisation lisible, pas une usine à gaz. Exemple idéal pour ce soir : un agent qui vérifie un sujet, résume ce qui compte, puis poste une alerte seulement si ça vaut le coup.

Trigger : tous les jours / toutes les heures / mercredi 20h
Agent : cherche, lit, compare, décide
Sortie : message Discord, synthèse, todo, follow-up ou silence
“Un bon cron doit économiser de l’attention, pas créer plus de bruit.”

21:05 - 21:25

3. La fin des tokens gratuits

Les modèles frontière vont coûter leur vrai prix. Les agents consomment plus que les chats classiques, et beaucoup de workflows ‘magiques’ deviennent vite chers quand ils tournent en continu.

Le piègeUn agent autonome peut lire, appeler des tools, réessayer, résumer, puis recommencer. Le compteur tourne.
La véritéLe bon modèle n’est pas toujours le meilleur modèle. C’est le moins cher qui fait correctement le job.
Le localLM Studio, Gemma, Qwen et autres modèles locaux deviennent une stratégie économique, pas juste idéologique.
Le routageFrontière pour les décisions critiques, local ou petit modèle pour les routines répétitives.
“Personne n’est prêt, parce que tout le monde a testé l’IA avec le prix subventionné.”
